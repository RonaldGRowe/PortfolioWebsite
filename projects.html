Streaming Tracker

Designed a C# Windows Form to log and store streaming content data for up to four users. Utilized multiple forms to be able to track movies, TV shows, and the unwatchable content. It stores whether you have watched them, want to watch, or don't want to watch. Autoplay always got me confused on which episode I was watching and combined with so many choices, I could not keep up with all the data. I was able to use text files to store the data for optimal read and save speeds for users. The download is a .exe file for Windows users.

C# | Windows Forms | Visual Studio | FileStream | StreamWriter/Reader

C# vs Python

Comparing two great programming languages by weighing the runtime differentials and the ease of using/writing the language. C# by far takes the runtime award with the ability to execute more tasks in a shorter amount of time. Python wins the ease of use award with a less verbose/lengthy syntax and so many useful modules to utilize. Funny to think some modules used to decrease Python runtimes are written in C. I enjoy writing code in Python, but the runtime factors can be a business deal breaker. This is understandable and why we have so many choices when it comes to programming languages.

C# | Python | Timers | Linux Ubuntu Terminal | Visual Studio
